It Will Take Time

Read the code and understand what is happening/should be happening at each step. Once you have a clear picture of what should the functionality should be, then begin writing code.

I also suggest writing your algorithms out in comments before running it. Gather your thoughts and make sure what you are doing makes sense.

Goal of the Game

Move around and the rid the plains of sneaky sneks and sloths. Even though the snakes and sloths never did anything to hurt us.

Rules of the Game

Pieces

  • Adventurer

    • Always moves first per iteration.
    • May move north, south, east or west. Up, down, right, left.
    • May not move diagonally.
    • Cannot move off of the map.
    • Invincible, though may take damage.
    • Never tires.

  • Log

    • Never moves. Never tires.
    • Can be tripped over (resulting in 1 damage to hitpoints).
    • Does nothing, other than exist.

  • Sneaky Snek

    • Moves three turns before tiring.
    • Does not move for two turns while resting.
    • Moves in a random direction.
    • May move north, south, east or west. Up, down, right, left.
    • May not move diagonally.
    • Can be killed if tired.
    • Can deal two damage to the adventurers hitpoints if not tired.

  • Sloth

    • Always tired.
    • Has a 1/3 chance of moving per turn.
    • May move north, south, east or west. Up, down, right, left.
    • May not move diagonally.
    • Can be killed if tired (always).
    • Too tired to deal damage to the adventure.

Actions

Each piece performs their action during each iteration. The adventurer must always go first.

View the act() method from the Plane class to understand what order the actions are taken in. There is a precondition that the Piece not act if has already acted. You should be able to discern as to why a Piece may potentially act more than once.

Log's action should be empty. They do not perform any action.

Note that for any movement by enemies the destination must be empty.

SneakySnek's need to keep track of the number of moves they make. A snek will move three times before becoming tired. The snek should move attempt to move in a random direction and the move counter should only be incremented in the event that the snek successfully moves. Once tired, the snek must rest for two moves during which it is vulnerable to attacks.

Sloth's are always tired so the boolean tired should never change. A sloth has a 1/3 chance of moving in a random direction.

The Adventurer can obtain the move selected by the user from the Plane class. The adventurer should try to move to a location, if that does not work, it should try to attack the location. If the move/attack succeeds, then the position should be updated.

Movement

When calculating the coordinates for movement in the act() methods, you should start with the x, y location the Piece is currently at. Save these values as fromX and fromY. Set toX and toY as the same value as fromX and fromY. Then update them according to the direction (note only one value changes per direction).

Example, if you're on 0,0 and you move SOUTH. Then the result should be 0,1.

Codebase

Overview

Class Purpose
AdventureTime The main game class. Runs the game.
Plane The plane contains all the pieces in the game and enforces most of the rules on the board.
Piece An abstract class describing the general functionality of all the pieces.
Adventurer, SneakySnek, Sloth, Log Extends Piece and implements the abstract methods and any additional requirements.

Details

AdventureTime

The AdventureTime class simply controls the game. It will ask the user for a name, display the current map, ask the user for a direction to take, and ask the Plane class to act. It will do this until the game ends or the player is exhausted (quits).

Plane

The Plane class models a two-dimensional plane on which the game will be played. You will notice this class has many fields. The plane field is a two dimensional array of Pieces. The size of this plane is determined by the class fields width and height. It is important we keep track of the adventurer piece, so another field is an object reference to our adventurer. There are also many constants declared for default values. Most of these are used in the Constructor methods.

One of the most important methods for this class is act(). This method tells all the pieces on the plane to perform their action. You should take the time to understand what is happening in this method.

The next methods you should focus on understanding are:

  1. isEmpty()
  2. isOnPlane()
  3. setupPlane()
  4. rand()
  5. randomDirection()
  6. move()
  7. toString()

You should start thinking about how the placePiece() and attackPieceAt() methods should work.

Piece

Piece is an abstract class that models what a Piece on the board should implement as a minimum. You will have to implement the fields needed. There will need to be a type, x and y coordinate, and booleans for whether the piece is tired and has acted yet.

Note that Piece is an abstract class so the act() method does not need to be implemented in Piece. It must be implemented by any subclass.
